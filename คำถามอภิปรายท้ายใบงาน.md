# คำถามอภิปรายท้ายใบงาน

## 1. เหตุใดจึงเลือกใช้ useReducer แทน useState ในการจัดการตะกร้าสินค้า

useReducer เหมาะสมกับ State ที่มีการเปลี่ยนแปลงหลายรูปแบบ เช่น การเพิ่มสินค้า ลดจำนวนสินค้า และลบสินค้าออกจากตะกร้า การใช้ Reducer ทำให้สามารถรวม Logic การจัดการ State ไว้ในที่เดียว ส่งผลให้โค้ดอ่านง่าย ดูแลรักษา และขยายระบบได้สะดวกกว่า useState เมื่อโปรเจกต์มีความซับซ้อนขึ้น

---

## 2. useMemo ช่วยเพิ่มประสิทธิภาพของระบบอย่างไร

useMemo ใช้สำหรับจดจำผลลัพธ์ของการคำนวณที่ใช้เวลาหรือทำซ้ำบ่อย ๆ เช่น การกรองรายการสินค้าและการคำนวณราคารวมของตะกร้า เมื่อข้อมูลที่เกี่ยวข้องไม่เปลี่ยน React จะนำค่าที่คำนวณไว้มาใช้ซ้ำ ทำให้ลดการประมวลผลที่ไม่จำเป็นและช่วยให้ระบบทำงานได้รวดเร็วขึ้น

---

## 3. useCallback มีประโยชน์ในสถานการณ์ใด และหากไม่ใช้จะเกิดผลอย่างไร

useCallback ใช้สำหรับจดจำฟังก์ชัน เพื่อไม่ให้ React สร้างฟังก์ชันใหม่ทุกครั้งที่ Component Render เหมาะกับการส่งฟังก์ชันผ่าน Props ไปยัง Child Component ที่ใช้ React.memo หากไม่ใช้ useCallback อาจทำให้ Child Component Render ใหม่โดยไม่จำเป็น ส่งผลให้ประสิทธิภาพลดลงเมื่อแอปพลิเคชันมีขนาดใหญ่

---

## 4. เปรียบเทียบการใช้ useContext กับการส่งข้อมูลผ่าน Props (Props Drilling)

| useContext | Props Drilling |
|------------|----------------|
| แชร์ข้อมูลให้หลาย Component ได้โดยตรง | ส่งข้อมูลผ่าน Props ทีละชั้น |
| ลดการส่ง Props หลายทอด | ต้องส่งผ่าน Component ที่ไม่จำเป็น |
| เหมาะกับข้อมูลส่วนกลาง เช่น ผู้ใช้หรือข้อมูลตะกร้า | เหมาะกับข้อมูลที่ใช้เฉพาะระหว่าง Parent และ Child |
| โค้ดดูแลรักษาง่ายเมื่อโปรเจกต์มีขนาดใหญ่ | เมื่อโปรเจกต์ใหญ่จะทำให้โค้ดซับซ้อนและแก้ไขยาก |

### สรุป

ในโปรเจกต์นี้เลือกใช้ useContext เพื่อจัดการข้อมูลตะกร้าสินค้า เนื่องจากหลาย Component เช่น Cart, CartItem และ Summary จำเป็นต้องเข้าถึงข้อมูลเดียวกัน การใช้ useContext ช่วยลดปัญหา Props Drilling และทำให้โครงสร้างของโปรแกรมมีความเป็นระเบียบมากขึ้น
