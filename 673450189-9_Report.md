# 1. ProductList.jsx

## หน้าที่

แสดงรายการเครื่องดื่มทั้งหมด โดยรับข้อมูลสินค้าผ่าน Props แล้ววนลูปสร้าง ProductCard ของแต่ละสินค้า

## หลักการทำงาน

- รับ products ผ่าน Props
- ใช้ map() วนลูปข้อมูลสินค้า
- ส่งข้อมูลสินค้าแต่ละตัวไปยัง ProductCard

## ตัวอย่างโค้ด

```jsx
products.map((product) => (
  <ProductCard
    key={product.id}
    product={product}
  />
));
```

## เหตุผลที่เลือกใช้

- ลดการเขียนโค้ดซ้ำ
- รองรับจำนวนสินค้าได้ไม่จำกัด

## ข้อดี

- โค้ดอ่านง่าย
- เพิ่มสินค้าง่าย
- แยกหน้าที่ชัดเจน

## ข้อจำกัด

- ต้องมี id ที่ไม่ซ้ำกัน

# 2. ProductCard.jsx

## หน้าที่

แสดงรายละเอียดของสินค้าแต่ละรายการ ได้แก่

- รูปภาพ
- ชื่อสินค้า
- ราคา
- หมวดหมู่
- ปุ่มเพิ่มลงตะกร้า

## หลักการทำงาน

เมื่อกดปุ่ม "เพิ่มลงตะกร้า"

จะ dispatch Action ไปยัง cartReducer

## ตัวอย่างโค้ด

```jsx
dispatch({
    type:"ADD",
    payload:product
})
```

## เหตุผลที่เลือกใช้

แยกการแสดงผลสินค้าออกจากหน้าหลัก ทำให้แก้ไขได้ง่าย

## ข้อดี

- นำกลับมาใช้ซ้ำได้
- แก้ไขง่าย

## ข้อจำกัด

- ต้องรับข้อมูลผ่าน Props หรือ Context

# 3. SearchBar.jsx

## หน้าที่

ใช้ค้นหาชื่อสินค้า

## หลักการทำงาน

เมื่อผู้ใช้พิมพ์ข้อความ

จะอัปเดต State search

แล้วนำไป Filter รายการสินค้า

## ตัวอย่างโค้ด

```jsx
onChange={(e)=>setSearch(e.target.value)}
```

## เหตุผลที่เลือกใช้

ช่วยให้ค้นหาสินค้าได้รวดเร็ว

## ข้อดี

- ใช้งานง่าย
- ตอบสนองทันที

## ข้อจำกัด

- ค้นหาเฉพาะชื่อสินค้า

# 4. CategoryFilter.jsx

## หน้าที่

กรองสินค้าแยกตามหมวดหมู่

- Coffee
- Tea
- Bakery
- Dessert

## หลักการทำงาน

เมื่อกดปุ่มหมวดหมู่

จะเปลี่ยนค่า category

และกรองข้อมูลใหม่

## ตัวอย่างโค้ด

```jsx
setCategory(item)
```

## เหตุผลที่เลือกใช้

ช่วยให้ผู้ใช้ค้นหาสินค้าได้ง่ายขึ้น

## ข้อดี

- ใช้งานง่าย
- ลดจำนวนสินค้าที่แสดง

## ข้อจำกัด

- ต้องกำหนดหมวดหมู่ล่วงหน้า

# 5. products.js

## หน้าที่

เก็บข้อมูลสินค้า

เช่น

- id
- name
- price
- category
- image

## ตัวอย่าง

```jsx
{
 id:1,
 name:"Espresso",
 price:60,
 category:"Coffee"
}
```

## เหตุผลที่เลือกใช้

แยกข้อมูลออกจาก Component

## ข้อดี

- เพิ่มสินค้าได้ง่าย
- แก้ไขสะดวก

## ข้อจำกัด

- ยังเป็นข้อมูลแบบคงที่
